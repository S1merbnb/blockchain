How It Works

A clear, step-by-step explanation of how weather data becomes auditable insurance events.

1

Data Collection

We aggregate multiple providers — OpenWeatherMap, WeatherAPI, NASA POWER — to improve coverage and reduce single-source errors.

2

Damage Detection Logic

Signals are evaluated and a confidence score is assigned. Visual color scale communicates severity (red → green).

3

Blockchain Logging

Events progress through states: Pending → Confirmed → Cancelled. Each state change is immutably recorded.

4

Human Validation

Insurers review flagged events before payouts. This preserves legal and ethical compliance while keeping automation transparent.

5

Notification

Farmers are notified only after confirmation. SMS and Email integrations are planned for future releases.
